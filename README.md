# Frontend Test Task (React + React Query)

## 📌 Описание

Необходимо реализовать SPA-приложение для работы с корзинами пользователей на основе публичного API DummyJSON.

Документация API:  
https://dummyjson.com/docs/carts

Приложение должно позволять просматривать список корзин, переходить в детали конкретной корзины и изменять данные корзины.

---

## 🛠 Tech Stack

Обязательно использовать:

- React (Vite)
- TypeScript
- @tanstack/react-query
- @emotion/styled
- Zustand или Context API

Допускается использование дополнительных библиотек (react-router, axios/fetch и др.).

---

## 📡 API

Использовать следующие эндпоинты:

### 1. Получение списка корзин

GET https://dummyjson.com/carts

Поддерживает параметры:
- limit
- skip

Пример:
https://dummyjson.com/carts?limit=10&skip=0

---

### 2. Получение одной корзины

GET https://dummyjson.com/carts/:id

Пример:
https://dummyjson.com/carts/1

---

### 3. Обновление корзины

PUT https://dummyjson.com/carts/:id

Пример body:

{
  "merge": true,
  "products": [
    {
      "id": 1,
      "quantity": 5
    }
  ]
}

---

# 🧩 Функциональные требования

## 1️⃣ Страница списка корзин

Отобразить список корзин со следующими данными:

- ID корзины
- userId
- количество товаров
- общая сумма
- кнопка перехода в детали

### Требования:

- Реализовать пагинацию (limit / skip)
- Отображать состояние загрузки
- Обрабатывать состояние ошибки
- При переходе между страницами поведение интерфейса должно быть корректным и предсказуемым
- При возврате со страницы деталей состояние списка должно сохраняться

---

## 2️⃣ Детальная страница корзины

Роут:
`/carts/:id`

Отобразить:

- ID корзины
- userId
- список товаров:
  - название
  - цена
  - количество
  - итог по товару
- общую сумму корзины

### Требования:

- Отображать состояние загрузки
- Обрабатывать состояние ошибки
- Интерфейс должен оставаться отзывчивым при навигации

---

## 3️⃣ Редактирование корзины

На странице деталей реализовать возможность:

- изменить количество товара
- удалить товар из корзины

После изменения данные должны корректно обновляться:

- на странице деталей
- в списке корзин

Обрабатывать возможные ошибки запроса.

---

## 4️⃣ Глобальный стейт

Через Zustand или Context реализовать хранение:

- текущей страницы пагинации
- выбранного limit
- (опционально) фильтрации по userId

Состояние должно сохраняться при навигации между страницами.
